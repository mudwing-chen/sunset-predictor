# 朝霞晚霞预测工具 — 设计文档

> 摄影师专用，预测任意地点未来 2 天的朝霞/晚霞质量，纯静态网页，PWA 可安装到手机。

## 1. 概述

为风光摄影师打造一个**零成本、无需注册、即开即用**的朝霞晚霞质量预测工具。输入地点（自动定位、搜索或地图点选），即可查看当天和次日的朝霞/晚霞评分及详细气象因子分析。

## 2. 技术栈

| 层 | 技术 | 原因 |
|---|------|------|
| 前端框架 | 纯 HTML + CSS + Vanilla JS | 无构建步骤，零依赖，文件即页面 |
| CSS 框架 | Tailwind CSS (CDN) | 快速实现响应式设计，无需构建 |
| 地图 | Leaflet + OpenStreetMap (CDN) | 轻量免费，支持地图选点 |
| 天文计算 | suncalc.js (CDN) | 计算日出日落/黄金时刻/方位角 |
| 气象数据 | Open-Meteo API | 免费、无 Key、全球覆盖、中国可用 |
| AQI 数据 | Open-Meteo Air Quality API | 同上 |
| PWA | Service Worker (手写) | 可安装到手机桌面 |
| 图标 | Lucide Icons (SVG/CDN) | 轻量、美观 |
| 字体 | 系统字体栈 | 零额外加载 |
| 部署 | Vercel / GitHub Pages | 免费 HTTPS 托管 |

## 3. 评分算法

基于多项学术资料综合设计：
- 美国专利 US 10459119 B2（日落色彩预测系统）
- 上海崇明朝霞晚霞观览气象指数（2025）
- Solyra 开源项目 AQI 评分体系
- 风光摄影师社区经验（DPReview）

### 3.1 评分因子与权重

| 因子 | 权重 | 数据来源 | 判定逻辑 |
|------|------|---------|---------|
| **☁️ 目标位置云量** | 30% | Open-Meteo 逐小时预报 | 中高云在 25%-60% 为最佳（散射光线）；<10% 太晴无云；>80% 太阴不透光 |
| **☀️ 太阳方向云偏移** | 20% | Open-Meteo + suncalc 方位角 | 基于云偏移法：判断从太阳照射方向到用户之间的云层是否阻挡光线。如果用户头顶云好但太阳方向阴天 → 大幅减分 |
| **🌫️ AQI 空气质量** | 15% | Open-Meteo Air Quality | AQI 50-100（轻度污染）→ 加分（颗粒增强色彩）；>150 → 减分（太浑浊）；<50 → 中性 |
| **👁️ 能见度** | 15% | Open-Meteo (visibility) | >10km → 加分；5-10km → 中性；<5km → 减分 |
| **🌧️ 降水加分** | 10% | Open-Meteo 逐小时预报 | 日出/日落前后 6 小时内有降水后转晴 → 大幅加分（雨后通透+丰富云层） |
| **⏰ 时间贴合度** | 10% | suncalc.js | 评分时间点与日出/日落时间越近得分越高，峰值在日出后/日落前 15-30 分钟 |

### 3.2 评分等级

| 分值 | 等级 | 颜色 | 建议 |
|------|------|------|------|
| 9-10 | 🌟 绝佳 | 金 | 必去，大概率出片 |
| 7-8.9 | 👍 很好 | 绿 | 推荐前往，值得早起 |
| 5-6.9 | 😐 一般 | 黄 | 碰运气，可去可不去 |
| 3-4.9 | 👎 较差 | 橙 | 不建议专门跑一趟 |
| 0-2.9 | 💔 很差 | 红 | 建议休息 |

### 3.3 云偏移法的前端简化实现

原专利的云偏移法需要完整的 3D 大气模型，在纯前端做近似简化：

1. 用 suncalc.js 计算出日出/日落时的 **太阳方位角**
2. 沿方位角方向偏移约 **100km**（低云）/ **200km**（中云）/ **350km**（高云）得到偏移点坐标
3. 额外请求 Open-Meteo 获取偏移点的云量数据（异步，不阻塞主流程）
4. 对照判断表：

   | 目标点云量 | 偏移点云量 | 评分影响 |
   |-----------|-----------|---------|
   | 疏云/碎云 | 晴朗/少云 | **维持高分** |
   | 疏云/碎云 | 疏云/碎云 | 略微降分 |
   | 疏云/碎云 | 阴天 | **大幅降分**（光线被挡） |
   | 晴朗 | 任何 | 本身已低分 |

5. 偏移点数据获取失败时不阻塞，降级为仅用目标点数据评分 + 标注"太阳方向数据暂缺"

### 3.4 自然语言评价

每个评分附带一句自然语言描述，例如：
- "云层适中，太阳方向晴朗，预计有浓烈色彩，非常推荐拍摄！"
- "头顶云量尚可，但太阳方向云层较厚，色彩可能不够鲜艳"
- "云量过少，天空可能比较平淡"
- "雨后转晴，空气通透，预计色彩丰富"

### 4.3 色彩方案

- **背景**: 深蓝灰 `#0f172a` → 渐变为深色 `#1e293b`（暗色模式，减少户外看屏眩光）
- **卡片**: 半透明深色 `rgba(30, 41, 59, 0.8)`
- **高亮/评分**: 暖金色 `#f59e0b`（与朝晚霞色调呼应）
- **文本**: 白色/浅灰
- **按等级着色**: 金/绿/黄/橙/红

## 5. 数据流

```
用户操作                   浏览器                          API
─────────               ────────                       ──────
打开网页 ─────→ 请求定位权限 ───→ 浏览器 Geolocation API
               ← 获取经纬度
               
输入城市 ─────→ 调用 Nominatim ──→ OpenStreetMap Geocoding
               ← 返回经纬度

点击地图 ─────→ Leaflet click event ──→ 获取点击坐标

               ──────────────────────────────────→ Open-Meteo Weather API
               ← JSON: 逐小时云量/降水/能见度
               
               ──────────────────────────────────→ Open-Meteo Air Quality API
               ← JSON: 逐小时 AQI

               调用 suncalc.js（本地计算，无网络请求）
               ← 日出/日落/黄金时刻/蓝色时刻时间

               运行评分算法（本地计算）
               ← 各时段评分 + 自然语言描述

               渲染 UI ← 更新 4 张卡片 + 详细因子
```

### 5.1 离线行为

PWA Service Worker 缓存页面资源（HTML/CSS/JS/图标），使得：
- 首次访问后，后续打开秒加载
- 无网络时显示缓存的最新数据（标记为"离线数据"）

## 6. 地图选点功能

### 6.1 交互方式

1. **打开地图**：点击顶部 🗺️ 按钮 → 展开全屏地图面板
2. **选点**：点击地图上任一点 → 出现标记 + 信息弹窗显示评分摘要
3. **确认**：点击"查看详情" → 关闭地图，更新所有卡片为该地点数据
4. **返回**：再次点击地图按钮 → 关闭地图

### 6.2 地图图层

- 基础图层：OpenStreetMap 地图
- **评分叠加层（可选）**：在当前视图范围内采样网格点，计算评分，用色块/等值线叠加显示——让用户一眼看出周围哪里评分高

## 9. 边界情况及错误处理

| 场景 | 处理 |
|------|------|
| 用户拒绝定位权限 | 显示搜索框，提示手动输入城市名 |
| 搜索不到地点 | 显示"未找到地点，请尝试输入 城市名,国家" |
| 网络请求失败 | 显示错误提示 + 重试按钮，PWA 模式显示缓存数据 |
| 浏览器不支持定位 | 降级到搜索框 |
| 某小时数据缺失 | 跳过该小时，用相邻小时插值 |
| 极地地区（极昼/极夜） | suncalc 返回无日出/日落 → 显示提示"该地区当前无日出/日落" |
| API 限流 | Open-Meteo 免费层无硬限流，但增加 1s 请求间隔保护 |
